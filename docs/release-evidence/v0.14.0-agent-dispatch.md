## Safe Antigravity missing-ID behavior

`go test ./internal/agentdispatch -run
'TestAntigravity(SuccessfulAnswerWithoutIDIsNotResumable|ResumeWithoutParsedIDRetainsDurableMapping|LogIDIsStrictAndVersionGateFailsLoudly)$'
-count=1` passed on this checkout. The fixtures prove that an absent or changed
Antigravity UUID retains the diagnostic log and reports `not resumable`; it
does not query global or provider-private session state. A later continuation
preserves an existing durable mapping only when its provider conversation ID is
still proven.

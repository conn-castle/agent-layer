## Safe missing-ID behavior

The v0.13 test coverage retains an Antigravity 1.1.1 fixture where the
conversation ID cannot be extracted. The fresh turn returns its answer with
`not resumable`, retains invocation evidence, and creates no mapping. The
extractor normalizes only Antigravity's documented logger prefix before matching
the semantic UUID lines; it does not read global latest-session state, search
prompts, or query private provider storage.

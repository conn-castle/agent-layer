## Safe missing-ID behavior

An earlier Antigravity 1.1.1 fresh probe used a log line shape that did not
match the initial exact extractor. It returned its answer successfully with
`not resumable`, retained the invocation log, and created no mapping. The
retained evidence showed the same two semantic UUID lines prefixed by
Antigravity’s standard logger metadata. v0.13 normalizes only that logger
prefix and then matches the two documented semantic lines exactly; it does not
read global latest-session state, search prompts, or query private provider
storage.
